本文章小白适用大佬请无视，但是小白也不能把它当成保姆指南，有些基础问题还是需要自己 Google 查询，比如：了解一个网站的基础构成。

前言

相信有许多互联网从业者希望有一个自己的个人网站（Blog）写写文章总结输出，但是苦于不会前端代码，学习成本太高等等原因一直拖着或者另寻出路，比如知乎专栏、微信公众号、简书、豆瓣等平台。

在这些平台上去分发自己的文章总结其实也挺好的，但是有一小搓“爱折腾的人”就是想拥有一个自己的个站，拥有掌控权。

选择平台

建站有许多非常成熟的平台，比如 Wordpress、Publii、Brizy、Ghost 等等资源非常丰富，但是有一定的学习成本，并且需要购买虚拟主机，域名备案、申请 https 证书等等繁复的前期准备。

对比几个平台后我选择 Gridea ，Gridea 是一个静态博客写作客户端，帮助你更容易地构建并管理博客或任何静态站点。它所有文件都在你的本地，无需管理数据库，有多个主题模版可以选择，自定义选项很丰富。

看完文章你可以不到一个小时就能建站，且不需要多少钱，不需要费力去维护，看完介绍你是不是已经思如泉涌、迫不及待、呼之欲出、扯扯犊子～

建站所需内容

  • 下载 Gridea
  • 自己习惯的 Markdown 编辑器，Gridea 包含编辑器但是我用不习惯，推荐Notion可导出 Markdown
  • 注册 Github
  • 购买独立域名：阿里云、腾讯云、 GoDaddy（如果你需要的话）

建站发布流程快速预览

  1. Gridea 设置好基础配置
  2. 把自己域名解析指向 id.github.io
  3. 测试 Gridea 远程链接
  4. 托管在你的 Github 仓库上面，通了即可输入你的域名看到网页了
  5. 把你写好你的文章粘贴到 Gridea 编辑器里发布
  6. 同步你的本地文件到 Github
  7. 记得备份你的 Gridea 本地文件到 Dropbox 或者其他云盘
  8. 完结撒花
